# 최단 경로 시뮬레이션 프로젝트

이 프로젝트는 그래프 이론의 핵심 알고리즘 중 하나인 **다익스트라(Dijkstra) 알고리즘**을 사용하여 주어진 그래프에서 두 노드 사이의 최단 경로를 찾는 시뮬레이션입니다. Java 언어로 구현되었으며, Maven 빌드 도구를 사용합니다.

## 프로젝트 개요

산업 현장에서 물류 최적화, 네트워크 라우팅, 내비게이션 시스템 등 다양한 분야에서 최단 경로 알고리즘은 매우 중요하게 활용됩니다. 본 프로젝트는 이러한 실용적인 알고리즘의 동작 방식을 이해하고 직접 구현해보는 것을 목표로 합니다.

## 코드 구조

프로젝트는 다음의 주요 파일들로 구성되어 있습니다.

*   `pom.xml`: Maven 프로젝트 설정 파일입니다. 프로젝트의 의존성 관리 및 빌드 설정을 포함합니다.
*   `src/main/java/com/example/pathfinding/`: Java 소스 코드가 위치하는 디렉터리입니다.
    *   `Node.java`: 그래프의 각 노드를 나타내는 클래스입니다. 노드의 이름, 시작점으로부터의 거리, 그리고 최단 경로 상의 이전 노드 정보를 저장합니다.
    *   `Graph.java`: 그래프 전체를 관리하는 클래스입니다. 노드들을 저장하고, 노드들 간의 간선(연결)을 추가하는 기능을 제공합니다. `Edge` 내부 클래스를 통해 간선의 목적지와 가중치를 정의합니다.
    *   `Dijkstra.java`: 다익스트라 최단 경로 알고리즘을 구현한 클래스입니다. 시작 노드로부터 모든 노드까지의 최단 거리를 계산하고, 경로를 역추적하여 출력하는 기능을 제공합니다.
    *   `Main.java`: 프로그램의 진입점입니다. 샘플 그래프를 생성하고, 다익스트라 알고리즘을 실행하여 그 결과를 콘솔에 출력합니다.

## 실행 방법

이 프로젝트는 Maven을 사용하여 빌드하고 실행할 수 있습니다.

1.  **빌드:** 프로젝트 루트 디렉터리(`C:\server\gpt-code-diary\codes\202604\26`)에서 다음 명령어를 실행하여 프로젝트를 빌드합니다.
    ```bash
    mvn clean install
    ```
    이 명령어는 필요한 의존성을 다운로드하고, 소스 코드를 컴파일하며, 실행 가능한 JAR 파일을 `target` 디렉터리에 생성합니다.

2.  **실행:** 빌드가 성공적으로 완료되면, 다음 명령어를 사용하여 프로그램을 실행할 수 있습니다.
    ```bash
    java -jar target/shortest-path-simulation-1.0-SNAPSHOT.jar
    ```
    프로그램이 실행되면 `Main.java`에 정의된 샘플 그래프에 대한 다익스트라 알고리즘의 최단 경로 계산 결과가 콘솔에 출력될 것입니다.

## 추가 정보

*   `Graph.java`의 `addEdge` 메서드는 현재 무방향 그래프를 가정하고 간선을 양쪽에 추가합니다. 방향 그래프가 필요한 경우 이 부분을 수정해야 합니다.
*   다익스트라 알고리즘은 음수 가중치를 가진 간선이 있는 그래프에서는 올바르게 동작하지 않습니다. 음수 가중치가 필요한 경우에는 벨만-포드(Bellman-Ford) 또는 SPFA와 같은 다른 알고리즘을 고려해야 합니다.
