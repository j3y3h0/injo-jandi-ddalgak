# 개인화 추천 시스템 (Personalized Recommendation System)

이 프로젝트는 간단한 **항목 기반 협업 필터링(Item-Based Collaborative Filtering)** 알고리즘을 사용하여 사용자에게 개인화된 항목을 추천하는 Java 애플리케이션이다. 데이터 기반 개인화 추천 시스템의 기본적인 동작 원리를 이해하는 데 목적이 있다.

## 프로젝트 설명

이 시스템은 사용자가 과거에 평가했던 항목들을 기반으로, 아직 평가하지 않은 항목들에 대한 사용자의 선호도를 예측하여 추천을 제공한다. 예를 들어, 특정 사용자가 A라는 영화를 높게 평가했고, 다른 많은 사용자들이 A 영화와 B 영화를 함께 높게 평가했다면, 이 시스템은 해당 사용자에게 B 영화를 추천할 가능성이 높다.

## 코드 구조

프로젝트는 다음의 Java 클래스들로 구성되어 있다:

*   **`Item.java`**:
    *   추천의 대상이 되는 개별 항목(예: 책, 영화, 음악 앨범)을 나타내는 클래스이다.
    *   `id`, `name`, `category` 등의 속성을 가지고 있다.
    *   `equals()`와 `hashCode()` 메서드가 `id`를 기준으로 오버라이드되어, `Map`의 키 등으로 사용될 때 항목의 고유성을 보장한다.

*   **`User.java`**:
    *   추천 시스템의 사용자를 나타내는 클래스이다.
    *   `id`, `name` 및 사용자가 각 `Item`에 부여한 `rating` (평가 점수) 맵(`Map<Item, Double>`)을 속성으로 가진다.

*   **`DataGenerator.java`**:
    *   추천 시스템 시뮬레이션을 위한 샘플 `Item` 및 `User` 데이터를 생성하는 유틸리티 클래스이다.
    *   실제 환경에서는 데이터베이스나 외부 API로부터 데이터를 로드하게 될 것이다.

*   **`CollaborativeFiltering.java`**:
    *   **항목 기반 협업 필터링 알고리즘**의 핵심 로직을 구현하는 클래스이다.
    *   두 `Item` 간의 유사도를 `코사인 유사도(Cosine Similarity)`를 이용하여 계산한다.
    *   대상 사용자가 평가하지 않은 `Item`에 대해 예측 평가를 계산하고, 가장 높은 예측 평가를 가진 `Item`들을 추천한다.

*   **`RecommendationSystem.java`**:
    *   이 애플리케이션의 메인 진입점 (`main` 메서드)이다.
    *   `DataGenerator`를 호출하여 샘플 데이터를 로드하고, `CollaborativeFiltering` 객체를 생성하여 특정 사용자 (`사용자 5`)에게 추천을 수행한 후 결과를 출력한다.

## 실행 방법

이 프로젝트는 표준 Java 환경에서 컴파일하고 실행할 수 있다.

1.  **자바 개발 키트(JDK) 설치**:
    *   시스템에 JDK 8 이상이 설치되어 있는지 확인한다. (명령 프롬프트 또는 터미널에서 `java -version` 실행)

2.  **소스 파일 준비**:
    *   모든 `.java` 파일을 한 디렉터리(`C:\server\gpt-code-diary\codes\202604\05`)에 저장한다.

3.  **컴파일**:
    *   저장된 디렉터리로 이동하여 다음 명령어를 사용하여 모든 Java 소스 파일을 컴파일한다.
    ```bash
    javac *.java
    ```
    *   성공적으로 컴파일되면 `.class` 파일들이 생성될 것이다.

4.  **실행**:
    *   컴파일이 완료된 후, 다음 명령어를 사용하여 `RecommendationSystem` 클래스를 실행한다.
    ```bash
    java RecommendationSystem
    ```
    *   콘솔에 샘플 데이터 로딩 과정과 "사용자 5"에 대한 추천 결과가 출력될 것이다.

## 주요 사용 알고리즘/기술

*   **협업 필터링 (Collaborative Filtering)**: 사용자 행동 데이터를 기반으로 추천을 생성하는 기법이다. 이 프로젝트에서는 `항목 기반(Item-Based)` 방식을 사용한다.
*   **코사인 유사도 (Cosine Similarity)**: 두 벡터의 유사도를 측정하는 방법이다. 이 시스템에서는 두 항목에 대한 사용자 평가 벡터 간의 유사도를 계산하는 데 사용된다. 값이 1에 가까울수록 두 항목은 유사하다고 판단된다.

이 프로젝트는 실제 산업 수준의 추천 시스템에 비해 매우 단순화된 형태이지만, 기본적인 추천 로직과 데이터 흐름을 이해하는 데 유용한 시작점이 될 것이다.
