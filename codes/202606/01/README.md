# 로그 분석 및 이상 감지 시스템 (C#)

## 프로젝트 개요

이 프로젝트는 애플리케이션 로그 데이터를 분석하고, 정의된 규칙에 따라 잠재적인 이상 징후를 감지하는 소규모 C# 콘솔 애플리케이션입니다. 실제 산업 환경에서 시스템 모니터링 및 문제 진단에 활용될 수 있는 기본적인 이상 감지 로직을 구현합니다.

## 코드 구조

*   `Program.cs`: 애플리케이션의 시작점입니다. 로그 처리기와 이상 감지기를 초기화하고, 샘플 로그 데이터를 생성하여 처리 과정을 보여줍니다.
*   `LogEntry.cs`: 단일 로그 엔트리(항목)의 구조를 정의하는 클래스입니다. 타임스탬프, 로그 레벨 (예: INFO, WARN, ERROR), 메시지 등의 속성을 포함합니다.
*   `LogProcessor.cs`: 로그 데이터를 처리하는 역할을 담당합니다. 여기서는 간단하게 문자열 형태의 로그 데이터를 `LogEntry` 객체 리스트로 변환하는 기능을 수행합니다. 실제 환경에서는 파일 읽기, 정규 표현식을 사용한 파싱 등이 포함될 수 있습니다.
*   `AnomalyDetector.cs`: 로그 엔트리 리스트를 분석하여 이상 징후를 감지하는 로직을 포함합니다. 이 예제에서는 특정 시간 동안 발생하는 `ERROR` 로그의 수가 임계값을 초과할 경우를 이상으로 판단하는 간단한 규칙을 사용합니다.

## 실행 방법

이 프로젝트는 .NET Core SDK가 설치된 환경에서 실행할 수 있습니다.

1.  **프로젝트 경로로 이동:**
    ```bash
    cd C:\server\gpt-code-diary\codes\202606\01
    ```
2.  **프로젝트 빌드:**
    ```bash
    dotnet build
    ```
3.  **애플리케이션 실행:**
    ```bash
    dotnet run
    ```

실행 시 콘솔에 샘플 로그 처리 결과와 감지된 이상 징후가 출력될 것입니다.
