# 실시간 로그 이상 탐지 시스템

## 1. 프로젝트 소개

이 프로젝트는 실시간으로 발생하는 로그 스트림에서 이상 징후를 탐지하는 소규모 시뮬레이션 시스템입니다. 예를 들어, 웹 서버나 애플리케이션에서 발생하는 로그를 분석하여, 평소와 다른 오류 발생 빈도나 특정 유형의 메시지 급증과 같은 이상 패턴을 식별하는 데 사용될 수 있습니다. 본 시스템은 C# 언어를 사용하여 구현되었으며, 실제 산업 환경에서 사용되는 기본적인 데이터 처리 및 분석 개념을 적용하였습니다.

## 2. 코드 구조

프로젝트는 다음 5개의 C# 파일로 구성됩니다:

*   `Program.cs`: 애플리케이션의 진입점입니다. 로그 생성, 처리, 이상 탐지 과정을 조율하고 결과를 출력합니다.
*   `LogEntry.cs`: 단일 로그 항목의 구조를 정의합니다. 타임스탬프, 로그 레벨, 메시지 등의 정보를 포함합니다.
*   `LogGenerator.cs`: 가상의 로그 데이터를 생성하는 역할을 합니다. 다양한 로그 레벨과 메시지를 무작위로 생성하여 실시간 스트림을 시뮬레이션합니다.
*   `LogProcessor.cs`: 원시 로그 문자열을 `LogEntry` 객체로 파싱하고, 필요한 경우 로그를 필터링하거나 정규화하는 로직을 포함합니다.
*   `AnomalyDetector.cs`: 로그 스트림에서 이상 징후를 탐지하는 핵심 로직을 구현합니다. 일정 시간 동안의 오류 로그 수나 특정 패턴의 발생 빈도 등을 모니터링하여 임계값을 초과할 경우 이상으로 판단합니다.

## 3. 실행 방법

1.  **빌드**: .NET SDK가 설치된 환경에서 프로젝트 루트 디렉터리(이 `README.md` 파일이 있는 곳)에서 다음 명령어를 실행하여 프로젝트를 빌드합니다:
    ```bash
    dotnet build
    ```
2.  **실행**: 빌드가 성공하면 다음 명령어를 사용하여 애플리케이션을 실행할 수 있습니다:
    ```bash
    dotnet run
    ```

애플리케이션은 가상의 로그를 생성하고, 이를 실시간으로 처리하여 이상 징후를 탐지한 후 그 결과를 콘솔에 출력할 것입니다.

## 4. 주요 기술 및 알고리즘

*   **객체 지향 프로그래밍 (OOP)**: C#의 클래스와 인터페이스를 활용하여 모듈화되고 확장 가능한 코드를 작성하였습니다.
*   **데이터 스트림 처리**: `LogProcessor`는 연속적인 로그 데이터를 처리하는 방식을 시뮬레이션합니다.
*   **슬라이딩 윈도우 (Sliding Window)**: `AnomalyDetector`는 특정 시간 범위 내의 로그 데이터를 분석하기 위해 개념적인 슬라이딩 윈도우 방식을 사용하여 이상 징후를 탐지합니다. (예: 지난 1분간의 오류 로그 수를 집계).
*   **임계값 기반 탐지 (Threshold-based Detection)**: 미리 정의된 임계값을 초과하는 로그 패턴을 이상 징후로 간주합니다. 이 프로젝트에서는 간단한 오류 로그 개수 임계값을 사용합니다.

이 프로젝트는 복잡한 시스템의 일부 구성 요소를 이해하는 데 도움이 되는 기본적인 예제입니다. 실제 시스템에서는 메시지 큐 (Kafka, RabbitMQ), 분산 처리 프레임워크 (Spark Streaming), 머신러닝 기반 이상 탐지 알고리즘 (Isolation Forest, ARIMA) 등이 활용될 수 있습니다.
