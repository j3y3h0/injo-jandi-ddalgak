# 실시간 센서 데이터 스트리밍 처리 시뮬레이션

## 프로젝트 개요

이 프로젝트는 실시간으로 발생하는 센서 데이터를 스트리밍하고, 이를 처리하여 이상 징후를 탐지하는 과정을 시뮬레이션합니다. 산업 현장이나 IoT 환경에서 수집되는 대량의 센서 데이터를 효율적으로 분석하고, 문제 발생 시 즉각적으로 대응하기 위한 기본적인 아키텍처를 JavaScript로 구현하였습니다.

## 파일 구조 및 역할

-   `main.js`:
    -   이 프로젝트의 진입점입니다.
    -   `dataGenerator`, `dataProcessor`, `anomalyDetector` 모듈을 사용하여 전체 데이터 흐름을 조정합니다.
    -   일정한 주기(1초)마다 새로운 센서 데이터를 생성하고, 이동 평균을 계산하며, 이상 징후를 탐지하여 콘솔에 출력합니다.

-   `dataGenerator.js`:
    -   가상의 센서 데이터를 생성하는 역할을 담당합니다.
    -   온도, 습도, 압력 등 임의의 값을 포함하는 센서 데이터 객체를 반환합니다. 실제 환경에서는 물리 센서나 외부 API로부터 데이터를 수신하는 부분에 해당합니다.

-   `dataProcessor.js`:
    -   수신된 데이터를 처리하는 로직을 포함합니다.
    -   주어진 데이터 배열에 대한 이동 평균을 계산하는 `calculateMovingAverage` 함수를 제공합니다. 이동 평균은 데이터의 추세를 파악하고 노이즈를 줄이는 데 사용되는 대표적인 통계 기법입니다.

-   `anomalyDetector.js`:
    -   처리된 데이터를 기반으로 이상 징후를 탐지하는 로직을 포함합니다.
    -   현재 값과 이동 평균 간의 차이가 특정 임계값을 초과하는 경우를 이상 징후로 판단하는 `detectAnomaly` 함수를 제공합니다. 이는 간단한 규칙 기반의 이상 탐지 알고리즘입니다.

## 실행 방법

1.  프로젝트 디렉터리로 이동합니다.
    ```bash
    cd C:\server\gpt-code-diary\codes\202602\09
    ```
2.  `main.js` 파일을 Node.js 런타임으로 실행합니다.
    ```bash
    node main.js
    ```

위 명령어를 실행하면 1초마다 새로운 센서 데이터가 생성되고, 이동 평균이 계산되며, 이상 징후가 탐지될 경우 경고 메시지가 콘솔에 출력되는 것을 확인할 수 있습니다.

## 사용된 라이브러리 및 알고리즘

-   **JavaScript (Node.js)**: 프로젝트 구현 언어 및 런타임입니다.
-   **이동 평균 (Moving Average)**: `dataProcessor.js`에서 구현된 통계 알고리즘입니다. 시계열 데이터의 추세를 부드럽게 하고 단기적인 변동을 제거하는 데 유용합니다.
-   **임계값 기반 이상 탐지 (Threshold-based Anomaly Detection)**: `anomalyDetector.js`에서 구현된 간단한 이상 탐지 방법입니다. 데이터가 미리 정의된 상한 또는 하한을 벗어날 때 이상 징후로 간주합니다.

이 프로젝트는 복잡한 시스템의 핵심 개념을 이해하고 테스트하기 위한 소규모 예시이며, 실제 운영 환경에서는 더 정교한 알고리즘과 견고한 아키텍처가 필요합니다.
