# 간단한 로깅 시스템 (Simple Logging System)

이 프로젝트는 데이터 분석 애플리케이션 등에서 활용할 수 있는 간단한 파일 기반 로깅 시스템을 C#으로 구현한 예제입니다. 비동기 처리를 통해 애플리케이션의 성능에 미치는 영향을 최소화하면서 로그를 파일에 기록하도록 설계되었습니다.

## 프로젝트 구조

프로젝트는 다음 4개의 C# 파일로 구성되어 있습니다.

*   **`ILogger.cs`**: 로거가 구현해야 할 기능을 정의하는 인터페이스입니다. `LogInfo`, `LogWarning`, `LogError`, `LogDebug`와 같은 메서드를 포함합니다. 이는 로깅 시스템의 확장성을 보장하며, 다양한 로깅 구현체로 쉽게 교체할 수 있도록 합니다.

*   **`LogEntry.cs`**: 개별 로그 메시지의 구조를 정의하는 데이터 모델 클래스입니다. 로그 시간 (`Timestamp`), 로그 수준 (`LogLevel`), 메시지 내용 (`Message`), 그리고 관련 예외 (`Exception`) 정보를 포함합니다. 또한, 로그 수준을 정의하는 `LogLevel` 열거형도 함께 포함되어 있습니다.

*   **`Logger.cs`**: `ILogger` 인터페이스를 구현하는 실제 로거 클래스입니다. 이 로거는 로그 메시지를 메모리 큐에 추가하고, 별도의 비동기 태스크를 통해 해당 큐의 메시지들을 지정된 로그 파일에 비동기적으로 기록합니다. 이를 통해 로그 기록 작업이 주 애플리케이션 스레드를 블록하지 않도록 합니다. 로그 파일은 애플리케이션이 실행되는 디렉터리 내의 `Logs` 폴더에 `application.log`라는 이름으로 생성됩니다.

*   **`Program.cs`**: 로깅 시스템의 사용 방법을 보여주는 메인 프로그램 파일입니다. `FileLogger` 인스턴스를 생성하고, 다양한 로그 수준(정보, 경고, 에러, 디버그)으로 메시지를 기록하는 예제를 포함합니다. 특히, 예외 처리 및 로거 종료 시 모든 대기 중인 로그가 기록되도록 `ShutdownAsync`를 호출하는 방법을 시연합니다.

## 실행 방법

이 프로젝트는 .NET Core 또는 .NET Framework 환경에서 빌드하고 실행할 수 있습니다.

1.  **프로젝트 빌드**:
    명령 프롬프트 또는 터미널을 열고, 이 프로젝트 파일들이 있는 디렉터리로 이동합니다.
    다음 명령어를 사용하여 프로젝트를 빌드합니다:

    ```bash
    dotnet build
    ```

    또는 Visual Studio를 사용하는 경우, Visual Studio에서 프로젝트를 열고 '빌드' 메뉴에서 '솔루션 빌드'를 선택합니다.

2.  **프로젝트 실행**:
    빌드가 성공적으로 완료되면, 다음 명령어를 사용하여 애플리케이션을 실행합니다:

    ```bash
    dotnet run
    ```

    Visual Studio를 사용하는 경우, F5 키를 누르거나 '디버그' 메뉴에서 '디버깅 시작'을 선택하여 실행할 수 있습니다.

3.  **로그 확인**:
    애플리케이션이 실행되면 콘솔에 메시지가 출력되고, 실행이 완료된 후에는 프로그램이 종료됩니다.
    로그 파일은 실행 파일이 위치한 디렉터리 내의 `Logs` 폴더(`C:\server\gpt-code-diary\codes\202602\23\Logs\application.log`)에 `application.log`라는 이름으로 생성됩니다. 이 파일을 열어 기록된 로그 메시지를 확인할 수 있습니다.

이 로깅 시스템은 산업에서 일반적으로 사용되는 비동기 로깅 패턴을 따르며, 구조화된 로깅과 예외 처리를 포함하여 실제 애플리케이션에 적용하기 용이하도록 설계되었습니다.
