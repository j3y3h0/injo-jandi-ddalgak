# 금융 이상 거래 탐지 시스템 (Java)

이 프로젝트는 AI 뉴스 토픽 중 하나인 '금융 사기 탐지'와 관련하여, Z-score라는 통계적 알고리즘을 사용해 사용자의 거래 패턴에서 벗어나는 이상 거래를 탐지하는 간단한 Java 애플리케이션이다.

## 프로젝트 구조

```
.
├── Main.java               # 애플리케이션의 주 실행 클래스
├── FraudDetector.java      # Z-score 기반 이상 탐지 로직을 포함하는 클래스
├── Transaction.java        # 거래 정보를 담는 데이터 모델 클래스
├── TransactionData.java    # 샘플 거래 데이터를 제공하는 유틸리티 클래스
└── README.md               # 프로젝트 설명 파일
```

### 주요 파일 설명

- **`Main.java`**: 전체 프로그램을 실행하고, 데이터 로드, 이상 탐지, 결과 출력의 흐름을 제어한다.
- **`FraudDetector.java`**: 핵심 로직이 담긴 파일이다. 사용자별로 거래 데이터를 그룹화한 뒤, 각 그룹의 평균과 표준편차를 계산한다. 이를 바탕으로 각 거래의 Z-score를 산출하여, 사전에 정의된 임계값(Threshold)을 초과하는 거래를 '이상 거래'로 식별한다.
- **`Transaction.java`**: 단일 거래의 속성(ID, 사용자 ID, 금액, 시간)을 정의하는 간단한 POJO(Plain Old Java Object)이다.
- **`TransactionData.java`**: 실제 데이터베이스나 파일 I/O 없이, 프로그램이 실행될 때 샘플 거래 데이터를 메모리에 생성해 제공하는 역할을 한다.

## 알고리즘: Z-score

Z-score는 특정 데이터 포인트가 전체 데이터셋의 평균으로부터 얼마나 떨어져 있는지를 표준편차 단위로 나타내는 통계적 척도이다. 수식은 아래와 같다.

> Z = (X - μ) / σ

- `X`: 개별 데이터 포인트 (여기서는 '거래 금액')
- `μ`: 데이터셋의 평균 (여기서는 '사용자의 평균 거래 금액')
- `σ`: 데이터셋의 표준편차 (여기서는 '사용자 거래 금액의 표준편차')

계산된 Z-score의 절댓값이 특정 임계값(보통 2, 2.5, 3)을 초과하면 해당 데이터는 통계적으로 이례적인 '이상치(Outlier)'로 간주될 수 있다. 이 프로젝트에서는 임계값을 `2.5`로 설정했다.

## 실행 방법

이 프로젝트는 외부 라이브러리 의존성이 없으므로, JDK(Java Development Kit)만 설치되어 있다면 간단히 컴파일하고 실행할 수 있다.

1.  **컴파일**:
    터미널 또는 명령 프롬프트에서 아래 명령어를 실행하여 모든 `.java` 파일을 컴파일한다.

    ```bash
    javac *.java
    ```

2.  **실행**:
    컴파일이 완료되면, `Main` 클래스를 실행한다.

    ```bash
    java Main
    ```
