# 실시간 로그 분석 및 이상 징후 감지 시스템 (Java)

## 1. 프로젝트 개요

이 프로젝트는 "실시간 로그 분석을 통한 이상 징후 감지"를 주제로 한 소규모 Java 애플리케이션입니다. 시스템에서 발생하는 다양한 로그 데이터를 분석하여 잠재적인 문제를 나타내는 '이상 징후'를 자동으로 감지하는 것을 목표로 합니다. 현재 구현에서는 `ERROR` 레벨 로그의 발생 빈도를 기반으로 이상 징후를 판단합니다.

## 2. 코드 구조

프로젝트는 다음 4개의 Java 파일로 구성되어 있습니다.

*   `LogEntry.java`: 개별 로그 항목의 데이터 구조를 정의합니다. 타임스탬프, 로그 레벨 (INFO, WARN, ERROR 등), 메시지를 포함합니다.
*   `LogProcessor.java`: 원시 로그 문자열을 파싱하여 `LogEntry` 객체로 변환하고, 이를 관리하는 역할을 합니다. 정규 표현식을 사용하여 로그 형식을 분석합니다.
*   `AnomalyDetector.java`: `LogEntry` 리스트를 분석하여 이상 징후를 감지하는 핵심 로직을 포함합니다. 현재는 `ERROR` 로그의 수가 특정 임계값을 초과하는 경우를 이상 징후로 판단하며, 로그 레벨별 분포 분석 기능도 제공합니다.
*   `Main.java`: 애플리케이션의 진입점입니다. 샘플 로그 데이터를 생성하고, `LogProcessor`를 통해 로그를 처리하며, `AnomalyDetector`를 사용하여 이상 징후를 감지하는 전체 과정을 시연합니다.

## 3. 실행 방법

이 프로젝트는 표준 Java 개발 환경(JDK 8 이상)이 설치된 환경에서 컴파일하고 실행할 수 있습니다.

### 3.1. 컴파일

프로젝트의 모든 `.java` 파일을 컴파일하기 위해 다음 명령어를 현재 디렉터리에서 실행합니다.

```bash
javac *.java
```

이 명령은 현재 디렉터리에 있는 모든 Java 소스 코드 파일을 컴파일하여 `.class` 파일을 생성합니다.

### 3.2. 실행

컴파일이 완료된 후, `Main` 클래스를 실행하여 로그 분석 및 이상 징후 감지 과정을 확인합니다.

```bash
java Main
```

콘솔에 샘플 로그 처리 결과와 이상 징후 감지 여부가 출력될 것입니다. `AnomalyDetector`에 설정된 임계값(`errorThreshold`)에 따라 다른 결과가 나타날 수 있습니다.

## 4. 특징 및 활용

*   **모듈화된 설계**: 각 기능(로그 엔트리, 처리, 감지)이 별도의 클래스로 분리되어 있어 유지보수 및 확장이 용이합니다.
*   **정규 표현식을 이용한 파싱**: `LogProcessor`는 유연한 로그 형식 파싱을 위해 정규 표현식을 활용합니다.
*   **간단한 임계값 기반 이상 징후 감지**: `AnomalyDetector`는 특정 로그 레벨(예: `ERROR`)의 발생 빈도를 기반으로 간단한 이상 징후를 감지합니다. 실제 시스템에서는 더 복잡한 통계적 또는 머신러닝 기반 알고리즘을 적용할 수 있습니다.
*   **산업적 활용 가능성**: 실제 운영 환경에서 서버 로그, 애플리케이션 로그 등을 모니터링하여 서비스 중단이나 성능 저하와 같은 문제 발생 가능성을 사전에 예측하고 대응하는 데 기초적인 아이디어를 제공할 수 있습니다.

## 5. 향후 개선 방향

*   **다양한 이상 징후 감지 알고리즘**: 시계열 분석, 머신러닝(예: Isolation Forest, One-Class SVM) 등을 도입하여 더 정교한 이상 징후를 감지합니다.
*   **실시간 데이터 스트리밍 연동**: Apache Kafka, Flink 등과 연동하여 실시간으로 유입되는 로그 스트림을 처리합니다.
*   **알림 기능**: 이상 징후 감지 시 Slack, 이메일, SMS 등으로 알림을 전송하는 기능을 추가합니다.
*   **설정 파일 관리**: 임계값과 같은 설정을 외부 파일(예: `config.properties`, YAML)에서 로드하도록 개선합니다.
